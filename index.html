Introduction

Portal de Pedido Liz - portalwebcmr.clientapp

Overview projeto

Bem vindo à documentação do Portal de Pedidos Liz!
Este portal é atualização de um projeto originalmente desenvolvido Maurício para sanar as necessidades internas dos representates e colaboradores da Liz Lingerie, nesta aplicação, são realizadas operações CRUD, no ERP SAP, utilizando API's para gravar e retornar informação.

Arquitetura e TechStack

As tecnologias utilizadas nesta aplicação são:

Vue.js(Vue2) versão 2.6.4, Vuex, C#, SAP, Webpack, Node.Js(versão > 14.0.0), e suas principais Libs de apoio são:
ExcelJs, vue-select, Boostrap-Vue, vue-router, ECharts, jqwidgets, vue-html2canva, vue-excel-export, dateFNS, vue-toasted, vue-sidebar-menu, entre outros localizados no package.json;

Acesso ao código fonte:

Ao clonar o projeto, verificar sua versão no Node ou Yarn, se assim preferir, para verificar rode:

  ```node -v ou yarn -v;

Caso sua versão do Node/Yarn esteja certa, rode um npm install --save-dev e espere a conclusão da instalação das depedencias do projeto.

Depois das estapas acima, rode npm run serve /yarn serve para rodar a aplicação localmente.

Servindo e distrubuindo a aplicação.

Para a servir a aplicação em ambiente local, lembre-se de antes, configurar o BackEnd e também  mongoDB/NOSQL Manager de acordo com o manual de instalação localizado na rede interna.

Como o mongoDB configurado, para servir a aplicação rode o Backend, logo após isso rode npm/yarn run serve, para que a aplicação rode em ambiente local.


Para distribuidor a aplicação em produtos, antes, verifique se os links de validação localizados no arquivos 'jwtinterceptor.js' e 'global.js' estão apontando para o ambiente local, caso esteja, aponte-os para o ambiente de produção.

Para gerar a build é recomendo que se instale o yarn, via npm install -g yarn, para evitar questões de cacheamento.

Limpe o seu gerenciador de pacotes escolhido, via yarn cache clean, e após isso monte a versão de distribuição com comando yarn build.
